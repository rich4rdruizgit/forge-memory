forge-memory

Persistent memory for AI agents. Decisions, bugs, patterns, and discoveries that survive across sessions.

Install GitHub

The Problem

AI agents forget everything between sessions. Every new conversation starts from zero. You re-explain the same decisions, the same architecture, the same conventions. Context is lost, mistakes repeat, and knowledge evaporates the moment a session ends.

The Solution

forge-memory is a persistent memory layer for AI agents. It stores observations — decisions, bugs, patterns, discoveries — in a local SQLite database with full-text search. Agents save and retrieve knowledge via MCP tools. Your context survives across sessions.

How It Works

Key Features

🔄

Semantic Upsert

The topic_key field prevents duplicates — saving to the same topic updates the existing observation instead of creating a new one.

🔍

FTS5 Search

Full-text search powered by SQLite FTS5 with BM25 scoring, recency ranking, and synonym expansion.

🏷️

Auto-Tagging

Keywords are automatically extracted and stored as tags, making observations discoverable without manual effort.

🕸️

Knowledge Graph

Typed relations between observations — extends, contradicts, replaces, depends_on — with traversal up to depth 3.

📋

Session Tracking

Open and close sessions with structured summaries so the next session picks up exactly where you left off.

⚒️

Forge Integration

Extract structured knowledge from spec and verify documents, search grouped by type, and get full feature context.

Quick Start

# Install
uv pip install forge-memory

# Add to Claude Code
claude mcp add forge-memory -- python -m forge_memory serve

# Restart Claude Code — done.
